HTML Host — 要件定義・概要設計

社内向けHTMLホスティング基盤。Claude Code で生成したHTMLを Google Drive にアップロードし、GAS Web App 経由で社内メンバーに配信する。

1. 要件定義

1-1. 背景・目的

項目内容
課題Claude Code で生成した HTML レポート・ダッシュボードを社内共有する手段がない（ローカルファイルを都度共有するのは非効率）
解決策GAS Web App + Google Drive で社内 HTML ホスティング基盤を構築し、CLI から1コマンドでデプロイ&共有可能にする
対象ユーザーSalesNow 社内メンバー（Google Workspace 認証必須）

1-2. 機能要件

#機能説明状態
F1HTML アップロードCLI（Claude Code）から HTML ファイルを Google Drive にアップロード。同名ファイルは上書き完了
F2ページ配信?page=xxx で指定ページの HTML をブラウザに配信完了
F3一覧表示Web App トップにデプロイ済みページ一覧を表示完了
F4ページ削除CLI または GAS エディタから指定ページを削除完了
F5deploy-html スキル/deploy-html で Claude Code から1コマンドデプロイ未着手
F6トークン自動リフレッシュOAuth トークン期限切れ時に自動でリフレッシュ完了

1-3. 非機能要件

項目要件
認証Web App は Google Workspace ドメイン認証（組織内のみ閲覧可）
セキュリティDrive API アクセスは OAuth 2.0 トークン。トークンファイルは .gitignore 対象
サイズ制限GAS HtmlService の上限: 約 500KB / ページ
可用性Google インフラ依存（SLA 99.9%）

1-4. スコープ外

除外項目理由
カスタムドメインGAS Web App はカスタムドメイン非対応
外部公開Workspace 管理者が外部アクセスを制限済み
doPost（HTTP POST）Workspace 管理者制限により curl からのアクセス不可。Drive API で代替

5. 認証・セキュリティ設計

認証フロー
経路認証方式スコープ備考
CLI → Drive API OAuth 2.0（.token.json） drive, script.external_request refresh_token で自動更新。GCPプロジェクト: 354900204969
ブラウザ → Web App Google Workspace SSO — salesnow.jp ドメインのみアクセス可
Web App → Drive GAS 実行ユーザー権限 drive デプロイ設定「自分として実行」
制約: Google Workspace 管理者設定により、GAS Web App への外部（匿名）アクセスは不可。そのため doPost（curl からの直接呼び出し）は使えず、Drive API 経由でアップロードする構成に変更。
